Git Integration for Jira Cloud provides indexing logs so you can inspect integration and repository synchronization failures in more detail than the on-screen error messages. Use this page when you need to diagnose indexing issues, verify a failed sync, or send the full log to support for analysis.

| Log action | Use when | Result | Notes |
| :--- | :--- | :--- | :--- |
| View log in the dialog | You need a quick look at recent indexing output | Partial log view in Jira | Good for fast triage |
| Download full log | You need the complete indexing record | Full log file on your machine | Best for deeper debugging or support cases |
| Send log to support | You need help analyzing the indexing failure | Support receives the log details | Include the downloaded full log when possible |

![](/wp-content/uploads/gij-gitcloud-actions-view-logs-2025.png)

## How to open the repository indexing log dialog

Access the repository indexing logs via ![](/wp-content/uploads/actions-icon.png) Actions ➜ **View log**. The Integration indexing log dialog appears.

![](/wp-content/uploads/gij-gitcloud-actions-view-logs-dialog-2025.png)

## How to use the partial log versus the full downloaded log

The dialog displays partial log information. Download the full log to save it to your local machine and view the entire log information using your preferred log viewer.

If you encounter issues, send this log to [gijsupport@gitkraken.com](mailto:gijsupport@gitkraken.com) for further analysis.
